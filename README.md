# Vibe Coding Guide

A practical workflow guide for designers who want to ship production frontend code with AI — built on real lessons from the **Tillett Homepage Project**.

## What's in here

[`Designer's-vibe-coding-guide.md`](./Designer's-vibe-coding-guide.md) — The main guide. Covers environment setup, how to work with Claude Code, prompt templates, Git basics for designers, and how to collaborate with engineers through PR reviews.

## Why this exists

This guide started as an experiment: could a designer ship real frontend code alongside an engineer, using Claude Code as the primary tool? The Tillett project was that experiment. This document captures what worked, what didn't, and the workflow we landed on.

## Who it's for

- **Designers** looking to try vibe coding on their own projects
- **Engineers** who want to understand how their designer counterpart is working, and how to set up a smooth review process together

## How to contribute

This is a living document — if something is technically inaccurate, missing context from the engineering side, or could be explained better, open a PR or leave a comment inline.

Things especially worth flagging:
- Anything that would raise a red flag in a real code review
- Steps that are unclear from a non-designer perspective
- Conventions or commands that differ across project setups

## Built with

- Workflow tested on: Next.js 16 + React 19 + TypeScript + Tailwind CSS v4 + shadcn/ui + Bun
- AI tooling: Claude Code
